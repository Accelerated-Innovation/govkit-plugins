# Feature: AI ticket summarization for support handoff (WI-5230)

| Role | What it means | Examples |
|---|---|---|
| **Generator** | Whatever produced Draft 0 | Aha! Feature Agent, an LLM prompt, a human author |
| **Tracker** | Wherever the feature fields live | Azure DevOps, Jira, Linear, a markdown file |
## Description

When a support ticket is escalated from Tier 1 to Tier 2, the system generates
an AI summary of the ticket history (customer messages, agent notes, prior
resolutions) so the Tier 2 engineer can get context without reading the full
thread. The summary appears at the top of the ticket for the receiving
engineer.

## Acceptance Criteria

```gherkin
Feature: AI ticket summarization on escalation

  Rule: Escalated tickets receive an AI-generated summary

    Scenario: Summary is generated when a ticket is escalated
      Given a Tier 1 agent escalates a ticket with 14 prior messages
      When the escalation completes
      Then an AI-generated summary appears at the top of the ticket
      And the summary is labeled "AI-generated summary"

    Scenario: Engineer can view the full thread alongside the summary
      Given an escalated ticket has an AI-generated summary
      When the Tier 2 engineer opens the ticket
      Then the full message history remains available below the summary

  Rule: Summarization failure does not block escalation

    Scenario: Escalation succeeds when summarization fails
      Given the summarization service is unavailable
      When a Tier 1 agent escalates a ticket
      Then the escalation completes
      And the ticket shows "Summary unavailable" instead of a summary
```

## NFRs

- Performance: summary is available within 30 seconds of escalation at the
  95th percentile. Evidence: performance test report. Owner: Engineering.
- Reliability: summarization failure never blocks or delays the escalation
  itself. Evidence: fault-injection test. Owner: QA.

## Evaluation Criteria

None provided.

## Assumptions

- Summaries are generated by the existing internal LLM gateway.

## Out of scope

- Summaries for tickets that are not escalated
- Customer-facing summaries
